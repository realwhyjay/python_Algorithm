## 1차 답안 (실패)

```python
num = int(input())

book = []
max = 0
bestseller = ""
for _ in range(num):
    book.append(input())

book_set = list(set(book[:]))

for i in book_set:
    if max < book.count(book_set[i]):
        bestseller = book_set[i]

print(bestseller)

```

문제가 되는 부분은 book.count(book_set[i])  
count() 안에는 직접 값을 넣어주여아하는데, 변수만 넣어도 되는 줄 알았으나.. 안되었다.  
book_set[i] 를 인식하지 못해서 일어난 일.   
방법도 모르겠어서 결국 답안을 참고하여 다시 풀어야했다.    

## 최종 답안 

```python
num = int(input())

# 나는 리스트를 사용하려고 하였는데, 답안에서는 딕셔너리를 사용했다.
books = {}

for _ in range(num):
    book = input()
		'''
    입력받은 책 이름이 books에 있는지 확인해주고
    없다면 해당 책 이름을 key로 가지는 원소에 value를 1로 설정해준다.
    있다면 value를 1씩 늘려준다
    '''
    if book not in books:
        books[book] = 1
        # books에 없다면 위와 같이 key와 value를 초기화해준다.
    else:
        books[book] += 1

'''
value의 값들만 리턴하는 values()를 사용 해준뒤
그 값들 중 가장 큰 값을 리턴하는 max를 사용하여
가장 많이 팔린 책을 파악한다.
'''
most_sell = max(books.values())

bestsellor = []

'''
key와 value를 쌍으로 리턴해주는 items()를 사용하여 for문을 돌려준다.
그중 value값인 number의 값이 most_sell과 같다면, 가장 많이 팔린 책이란 의미이기 때문에
그 경우에만 bestsellor에 해당 key값인 book을 append 해준다.
'''
for book, number in books.items():
    if number == most_sell:
        bestsellor.append(book)
        

# 같은 value를 가지는 key값이 있다면 사전순으로 정렬해주어야하기에 정렬을 해준 값중 가장 앞 값을 print해준다
print(sorted(bestsellor)[0])

```



## 또 다른 답안

```python
from collections import Counter

num = int(input())
books = []
for _ in range(num):
  books.append(input())
  
books.sort() # 문자열을 사전 순으로 정렬
cnt = Counter(books)

print(cnt.most_common(n=1)[0][0]) # 가장 많은 개수의 데이터의 key 출력
```

이번 문제처럼 데이터의 개수를 셀 때는 딕셔너리 타입이 많이 사용되는데,  
collections 모듈의 Counter 클래스를 사용하면 해당 코드를 한줄로 줄일 수가 있다.  

Counter 클래스에는 , 데이터의 개수가 많은 순으로 정렬된 배열을 리턴해주는 most_common이라는 메소드를 제공한다.  

### 예시 

```python
from collections import Counter

Counter('hello world').most_common() 
# [('l', 3), ('o', 2), ('h', 1), ('e', 1), (' ', 1), ('w', 1), ('r', 1), ('d', 1)]

# 메소드의 인자로 숫자를 넘기면 그 숫자만큼만 리턴한다.
# 위의 해답에서는 (n=1)로 넘겨줬는데 같은 의미이다.
Counter('hello world').most_common(1) 
# [('l', 3)]
```

하지만 해당 모듈을 사용한 경우 코드를 짤 때 편하고, 알아보기 편하다는 장점이 있지만.   
최종답안에 비해 메모리와 시간이 조금 더 나오기에 더 효율적으로 작동하는 코드는 아닌 것 같다.
